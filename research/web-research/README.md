# Web Research

Deep research across the web — searches multiple angles, cross-checks claims, and delivers a structured, source-cited summary.

## What It Does

Researches any topic by searching from at least three angles:
1. **Official sources** — docs, whitepapers, company announcements
2. **Community examples** — forums, repos, real-world usage
3. **Recent discussions** — news, social media, HN/Reddit threads

It cross-checks important claims across multiple sources, separates facts from interpretation, and marks uncertain claims explicitly.

## How to Use

Ask a research question and specify what you need: a competitor analysis, technical investigation, market overview, or source-backed summary. The more specific the question, the sharper the result.

## Output Example

```
## Key Findings
- React Server Components are stable in Next.js 14+, with 40% bundle reduction in production benchmarks (Vercel, 2024)
- Streaming SSR adoption among top 10k sites grew from 3% to 18% in 2024 (HTTP Archive)
- Unverified: claims of 2x performance improvement vary by app structure

## Sources
- https://nextjs.org/docs/app/building-your-application/rendering/server-components
- https://github.com/vercel/next.js/discussions/54321
- https://httparchive.org/reports/state-of-the-web

## Recommendation
- Migrate page-level data fetching to RSC for immediate bundle wins
- Hold off on full streaming migration until Q2 — community patterns still stabilizing
```

## Quality Standards

- Every factual claim links to a source URL
- Uncertain or single-source claims are marked as unverified
- Recent sources are preferred for fast-moving topics
- Final output is structured for decision-making, not just information gathering
## Why Use This Skill

Web research without structure produces link dumps, not insights. This skill enforces a methodology — search from multiple angles, cross-check claims, cite every source. The output is built for decision-making, not just information gathering.
